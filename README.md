# AI Solar Forecast - גרסה גנרית רב-מדינתית

## מה יש כאן

תיקייה עם קוד שמחליף/משדרג את `train_solar_model.py` המקורי שלך, כך שאפשר
להוסיף מדינות נוספות למודל בקלות (לא רק ספרד).

```
solar_forecast_generic/
├── country_config.py          # רשימת המדינות + נתונים עליהן (קו רוחב, קיבולת וכו')
├── data_loading.py             # קוד שקורא את קבצי הנתונים הגולמיים
├── build_dataset.py            # ממזג הכל לדאטהסט אחד עם features
├── train_model.py              # מאמן את המודל (מריצים את זה כרגיל)
├── leave_one_country_out.py    # בדיקת הכללה - איך המודל מתפקד על מדינה שלא ראה
└── data/
    ├── ES_solar_generaition_1_.csv        # נתוני ייצור ספרד (כבר יש לך)
    └── open-meteo-37_36N5_98W21m__1_.csv  # נתוני מזג אוויר ספרד (כבר יש לך)
```

## איך להריץ (שלב ראשון - רק ספרד, לוודא שהכל עובד)

1. **התקנת ספריות** (פעם אחת בלבד, בטרמינל/CMD בתוך התיקייה):
   ```
   pip install pandas numpy xgboost scikit-learn matplotlib
   ```
   (אם יש לך כמה גרסאות פייתון כמו שהיה בעבר, יכול להיות שתצטרכי `py -m pip install ...` במקום `pip install ...`)

2. **הרצת האימון:**
   ```
   python train_model.py
   ```
   (או `py train_model.py` אם `python` לא מזוהה)

3. **מה אמור לקרות:** תוך כמה שניות יודפסו תוצאות (RMSE, MAE, MAPE) - אמורות
   להיות כמעט זהות למה שקיבלת בגרסה הקודמת (RMSE כ-0.14). זה סימן שהריפקטור
   לא שבר כלום, רק סידר את הקוד להיות גנרי.

   ייווצרו גם שני קבצים חדשים: `model_evaluation.png` (גרף) ו-`xgb_solar_model.json` (המודל השמור).

## איך להוסיף מדינה נוספת (למשל גרמניה)

זה כרגע **לא יעבוד עדיין** כי חסר לך קובץ מזג אוויר לגרמניה. השלבים:

1. **להוריד נתוני מזג אוויר לגרמניה מ-Open-Meteo** (כמו שעשית לספרד), לקואורדינטות
   ברלין (52.52, 13.41) או כל מיקום אחר שתבחרי בגרמניה. לשמור את הקובץ בתיקיית `data/`.

2. **לעדכן את `country_config.py`** - לשנות את השורה:
   ```python
   "weather_file": "open-meteo-berlin.csv",
   ```
   כך שהשם יתאים בדיוק לשם הקובץ שהורדת.

3. **לוודא שיש לך גם קובץ ייצור לגרמניה** - אותו קובץ OPSD/ENTSO-E, רק עם
   עמודת `DE_solar_generation_actual` (בדיוק כמו שיש לך היום `ES_solar_generation_actual`).

4. **לערוך את `train_model.py`** - להוריד את ה-`#` משורת גרמניה:
   ```python
   COUNTRY_FILES = {
       "ES": "data/ES_solar_generaition_1_.csv",
       "DE": "data/DE_generation.csv",   # למחוק את ה-# מפה, לעדכן את הנתיב
   }
   ```

5. **להריץ שוב** `python train_model.py` - עכשיו זה יאמן על שתי המדינות יחד.

## איך להריץ את בדיקת ה-LOCO (Leave-One-Country-Out)

אחרי שיש לך לפחות 2 מדינות מוגדרות (כמו בסעיף הקודם), לעדכן גם את
`COUNTRY_FILES` בתוך `leave_one_country_out.py` (אותו עדכון בדיוק), ואז:
```
python leave_one_country_out.py
```
זה יאמן על כל מדינה חוץ מאחת, ויבדוק כמה טוב המודל מנבא על המדינה שהוא
מעולם לא ראה - זו הבדיקה האמיתית לכך שהמודל "מבין" את הקשר הכללי ולא
רק משנן דפוסים ספציפיים למדינות שהוא אומן עליהן.

## הערה חשובה - נתוני קיבולת לגרמניה

הם **הערכה גסה** שמצאתי בחיפוש מהיר - כדאי לאמת אותם מול המקור המדויק לפני
שסומכים על התוצאות: https://ourworldindata.org/grapher/installed-solar-pv-capacity
